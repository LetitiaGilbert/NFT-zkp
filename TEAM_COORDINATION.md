# Frontend Setup & Team Coordination Guide

## For the Frontend Developer (You)

### Step 1: Install Dependencies
```bash
cd frontend
npm install
```

### Step 2: Wait for Artifacts from ZKP Engineer
Ask your ZKP engineer to provide:
- [ ] `circuit.wasm` (Circom compiled output)
- [ ] `circuit_final.zkey` (Groth16 trusted setup)
- [ ] `verification_key.json` (from snarkjs key export)

Once you have these files, move them to `frontend/public/`

### Step 3: Get Contract Info from Blockchain Engineer
Ask your blockchain engineer to provide:
- [ ] Deployed verifier contract address (e.g., `0x123...`)
- [ ] Network you're deploying on (e.g., Sepolia)
- [ ] Contract ABI (JSON)

Update `src/utils/constants.js` with:
```javascript
export const VERIFIER_ADDRESS = '0x...'; // From blockchain engineer
export const NETWORK_ID = 11155111;      // From blockchain engineer
```

### Step 4: Start Development
```bash
npm run dev
```

Visit http://localhost:3000

## For the ZKP Engineer (Circom)

### What We Need From You
After you finish circuit development and testing:

1. **circuit.wasm**
   - Generated by: `circom circuit.circom --r1cs --wasm`
   - Location: `zkp/circuit_js/circuit.wasm`
   - Size: Usually 1-10MB depending on circuit complexity

2. **circuit_final.zkey**
   - Generated by: `snarkjs groth16 setup ...` (trusted setup)
   - Location: `zkp/circuit_final.zkey`
   - Size: Usually 50MB+

3. **verification_key.json**
   - Generated by: `snarkjs zkey export verificationkey circuit_final.zkey verification_key.json`
   - Location: `zkp/verification_key.json`
   - Size: A few KB

### How to Share
Copy these files to the frontend:
```bash
cp circuit_js/circuit.wasm ../frontend/public/
cp circuit_final.zkey ../frontend/public/
cp verification_key.json ../frontend/public/
```

### What the Frontend Does With These
- **circuit.wasm**: Executes circuit logic in the browser
- **circuit_final.zkey**: Generates Groth16 proofs
- **verification_key.json**: Used for local verification (optional)

Your circuit inputs are hardcoded in `ProofGenerator.jsx`:
```javascript
const inputs = {
  ownerHash: '123',
  expectedOwnerHash: '123',
  collectionId: '456',
  expectedCollectionId: '456'
};
```

**If your circuit has different signal names**, let the frontend developer know and they'll update the form.

## For the Blockchain Engineer (Solidity)

### What We Need From You

1. **Contract Address**
   - Where you deployed the verifier contract
   - Example: `0x5FbDB2315678afccb333f8a9c12c1fb3418e2c0a`
   - Network: Must tell us which network (Sepolia, Mainnet, etc.)

2. **Verifier Function Signature**
   - Your contract's proof verification function
   - Currently expected to be: `verifyProof(uint256[8] proof, uint256[] input) → bool`
   - If different, let frontend dev know and they'll update `contract.js`

3. **Network Details**
   - Chain ID (Sepolia = 11155111, Mainnet = 1)
   - RPC URL (optional, for fallback)

### How to Share
1. Give contract address to frontend developer
2. Provide contract ABI JSON
3. Tell them the network chain ID

### Contract Expectations
The contract should have a function like:
```solidity
function verifyProof(uint256[8] memory proof, uint256[] memory input)
    public pure returns (bool)
{
    // Verify the proof and return true/false
}
```

The frontend will call this function with:
- **proof**: Groth16 proof formatted as `[a, b, c]`
- **input**: Public signals from the circuit

### Day 1: Kickoff
- Frontend: Start frontend setup, wait for artifacts
- ZKP Eng: Finish circuit, compile it

### Day 2: Artifacts Ready
- ZKP Eng: Provide wasm, zkey, verification_key.json

## Testing Locally (Before Blockchain Deployment)

1. **Circuit is compiled** ✓
2. **Artifacts in public/** ✓
3. **Run `npm run dev`** ✓
4. **No contract yet?**
   - Frontend will error when trying to verify
   - But proof generation will work!
   - Use this to test ZKP side independently

5. **Mock contract for testing:**
   - Blockchain engineer can deploy simple verifier first
   - Or use a mock contract that always returns true
   - Frontend will work with any contract matching the interface

## Troubleshooting Checklist

### "circuit.wasm not found"
- [ ] Do you have files in `frontend/public/`?
- [ ] Is it named exactly `circuit.wasm`?
- [ ] Check: `ls frontend/public/`

### "Proof generation fails"
- [ ] Check browser console for detailed error
- [ ] Make sure wasm file is correct
- [ ] Try with simpler circuit inputs (e.g., "1", "1")

### "Verifier contract address not configured"
- [ ] Update `src/utils/constants.js` with real address
- [ ] Ask blockchain engineer for deployed address

### "Wrong network error"
- [ ] Check NETWORK_ID in constants.js
- [ ] Make sure MetaMask is on same network
- [ ] Use Sepolia testnet for development

### "On-chain verification fails"
- [ ] Check contract address is correct
- [ ] Check contract ABI matches
- [ ] Verify contract function name (likely `verifyProof`)
- [ ] Check blockchain engineer's contract compiles/deploys
